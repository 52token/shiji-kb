## 年表类文本的标注特点

### 1. 高频时间表达

年表章节的时间表达极其频繁,包括:

- **在位年数**: 简单数字(如"七"、"八")
- **纪年**: `〖%X年〗`、`〖%X月〗`
- **时长**: 需要消歧为 `〖%X岁/年|时长〗`

### 2. 时长消歧的判断标准

**需要消歧的情况**:
- 任职时长: `为/担任 + 职位 + X岁/年`
- 从属时长: `从/属/侍 + 人名 + X岁/年`
- 时间间隔: `绝/除 + X岁/年 + 后续事件`
- 持续状态: `守/系狱 + X岁/年`

**无需消歧的情况**:
- 纪年年号: `〖%六年〗`、`〖%元年〗`
- 月份: `〖%十月〗`
- 干支纪日: `〖%甲申〗`

### 3. 特殊表达

**"X岁 + Y月"组合**:
- 第一个表示时长,需消歧: `〖%二岁|时长〗〖%十月〗`
- 第二个表示月份,无需消歧

## 修正示例

### 类型1: 任职时长

```markdown
原文: 为〖;丞相〗〖%五岁〗
修正: 为〖;丞相〗〖%五岁|时长〗
说明: 担任丞相五年
```

### 类型2: 从属时长

```markdown
原文: 属〖@项梁〗，为楚〖;柱国〗。〖%四岁〗，〖@项羽〗死
修正: 属〖@项梁〗，为楚〖;柱国〗。〖%四岁|时长〗，〖@项羽〗死
说明: 跟随项梁四年后项羽死
```

### 类型3: 时间间隔

```markdown
原文: 国除，绝，〖%七岁〗。
修正: 国除，绝，〖%七岁|时长〗。
说明: 国除后间隔七年
```

### 类型4: 持续状态

```markdown
原文: 久〖[系狱〗〖%三岁〗
修正: 久〖[系狱〗〖%三岁|时长〗
说明: 被关押三年
```

## 标注质量评估

### 优点

1. **实体标注完整**: 人名、地名、官职、数量标注准确
2. **结构规范**: 表格格式保持完整
3. **纪年标注**: 年号、月份、干支纪日标注规范
4. **整体质量高**: 021-022章无需修正

### 改进方向

1. **时长消歧系统性**: 需要对"任职时长"、"从属时长"进行系统审查
2. **一致性**: 同类表达应统一使用消歧模式
3. **重点章节**: 018、020章因内容详细,需更细致的审查

## 规律总结

### 规律1: 表格式年表的时间密度

年表类文本的时间表达密度远高于叙事性文本:
- 叙事性文本: 平均每100行约5-10处时间表达
- 年表类文本: 平均每100行约50-100处时间表达

### 规律2: 时长表达的语境判断

判断是否需要消歧,关键在于语境:
- **纪年语境**: "X年,发生了..." → 无需消歧
- **持续语境**: "做了X年后..." → 需要消歧

### 规律3: 功臣表 vs 王子表

- **功臣侯者表**: 详细记录履历,时长表达多
- **王子侯者表**: 简单记录世系,时长表达少

## 建议

### 对标注者

1. 处理年表时,特别注意"X岁/年"表达
2. 遇到"为/从/属/守/系狱 + X岁/年",优先考虑时长消歧
3. 表格式文本可逐行检查,避免遗漏

### 对审核者

1. 年表类章节应作为重点审查对象
2. 建议使用正则表达式批量检查: `〖%[^〗]+岁(?!〗)`
3. 对018、020这类详细年表,建议多轮审查

### 对工具开发

1. 可开发自动检测工具,识别可能需要消歧的时长表达
2. 基于语境的自动消歧建议
3. 批量验证工具,确保修正后的完整性
