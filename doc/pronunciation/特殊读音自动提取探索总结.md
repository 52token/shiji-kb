# 史记特殊读音提取分析报告

## 一、任务概述

系统性扫描《史记》130篇及其注释（史记正义、史记集解），提取特殊读音词汇，以补充现有的特殊读音词典。

## 二、数据来源

1. **原文文件**：3个史记原文文件（archive/史记*.txt，排除正义、集解、索隐）
2. **注释来源**：
   - 史记正义.txt (3754行)
   - 史记集解.txt (2675行)
3. **现有词典**：docs/data/special-pronunciation.json (61个词条)

## 三、提取方法

### 3.1 尝试的方法

#### 方法1：基本模式匹配 (extract_pronunciation.py)
- 模式：`([一-龥]{1,3})音([一-龥]{1,4})`
- 结果：正义1048条，集解1070条
- **问题**：噪音极大，包含大量引用格式词如"徐广曰音"、"汉书音义曰"

#### 方法2：改进过滤 (extract_pronunciation_v2.py)
- 加入过滤词列表
- 结果：正义713词，集解573词，合并1209词
- **问题**：仍有大量噪音，如"徐广曰"、"韦昭曰"等

#### 方法3：学者注释提取 (extract_pronunciation_v3.py)
- 从【学者曰...音...】格式中提取
- 结果：正义3词，集解212词
- 特殊格式（如"头曼【韦昭曰音瞒】"）：集解37条

#### 方法4：原文频率验证 (analyze_pronunciation_candidates.py)
- 在史记原文中验证候选词出现频率
- 筛选条件：≥3次出现
- 结果：131个候选词（出现≥3次）
- **发现**：大部分高频词是噪音（常用字如"也"、"于"、"与"等）

### 3.2 核心发现

**注释格式复杂性**：
史记集解和正义的注释采用多层嵌套的学者引用格式，例如：

```
单于曰头曼【韦昭曰音瞒】
```

或

```
郦食其【郑徳曰音厯异基】
```

这种格式导致简单的正则表达式难以准确提取，同时又产生大量误匹配。

## 五、挑战与限制

### 5.1 数据质量问题

1. **引用格式噪音**：注释文本充满学者互引，如"徐广曰一作..."、"骃案汉书音义曰..."
2. **上下文缺失**：很多读音注释依赖上下文才能确定被注对象
3. **多音字干扰**：常用字的多音注释（如"为"、"中"、"行"）难以区分是否为特殊读音

### 5.2 技术限制

1. **正则表达式局限**：嵌套复杂的古文注释格式难以用简单模式匹配
2. **自动分类困难**：无法自动判断词汇类型（人名、地名、官职等）
3. **频率阈值问题**：高频词多为噪音，低频词可能遗漏重要专名

## 六、建议后续方案

### 6.1 人工审核方案

由于自动提取噪音过多，建议采用半自动方案：

1. **种子词扩展**：从现有61个词典词条中，找出它们在正义/集解中的注释模式
2. **相似模式检索**：根据种子词的注释格式，寻找相似模式
3. **人工审核确认**：对提取结果进行人工筛选和分类

### 6.2 针对性提取

重点关注以下几类：

1. **匈奴专名**：
   - 单于名：冒顿、头曼、呼韩邪等
   - 官职：阏氏、左右贤王等

2. **西域地名**：
   - 国名：大宛、月氏、乌孙等
   - 地名：葱岭、天山等

3. **复音官职**：
   - 仆射、大行、廷尉等

4. **特殊姓氏**：
   - 复姓：令狐、东郭、南宫等
   - 特殊读音姓：员（yùn）、区（ōu）、仇（qiú）等

### 6.3 数据整理方案

建议的JSON格式：

```json
{
  "text": "词汇",
  "pinyin": ["拼音数组"],
  "context": "类别（人名/地名/官职等）",
  "note": "注释来源和说明",
  "occurrences": "在原文中出现次数",
  "source_annotation": "原始注释文本",
  "scholar": "注释学者"
}
```

## 七、结论

1. **自动提取困难**：由于史记注释的复杂格式，纯自动提取会产生90%以上的噪音
2. **有价值发现**：从史记集解成功提取37条高质量专名读音
3. **需要人工介入**：建议采用半自动方案，结合人工审核
4. **现有词典价值**：61条人工整理的词条质量远高于自动提取
