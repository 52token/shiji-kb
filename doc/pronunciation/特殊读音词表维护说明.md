# 特殊读音词表维护说明

**版本**: 2.0
**日期**: 2026-03-31
**词表文件**: `docs/data/special-pronunciation.json`

## 概述

史记中包含大量古代人名、地名、族群称号等特殊读音词汇，这些词汇的读音与现代常用读音不同。本文档说明如何维护特殊读音词表，确保拼音注释准确。

**版本2.0更新**：已合并人工整理词表和《史记正义》提取词表，统一为单一文件。来自《史记正义》的词条在note字段中标注`[史记正义]`前缀。

## 词表结构

### 数据格式

```json
{
  "$schema": "特殊读音词表 v2.0",
  "description": "史记特殊读音词汇（人工整理 + 史记正义提取）",
  "version": "2.0",
  "lastUpdated": "2026-03-31",
  "note": "合并人工整理词表和史记正义提取词表，来自正义的词条在note中标注",
  "entries": [
    {
      "text": "冒顿",
      "pinyin": ["mò", "dú"],
      "context": "匈奴单于名",
      "note": "冒顿单于，匈奴历史上著名单于，'冒'读mò，'顿'读dú"
    },
    {
      "text": "阏与",
      "pinyin": ["yān", "yǔ"],
      "context": "地名",
      "note": "[史记正义] 赵奢破秦军处，正义：'音焉与'，'阏'读yān，'与'读yǔ"
    }
  ]
}
```

**字段说明**：
- `note`字段中带`[史记正义]`前缀的词条来自《史记正义》提取

### 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `text` | string | ✓ | 词汇原文（2-4字为宜） |
| `pinyin` | array[string] | ✓ | 每个字的拼音数组，必须与text字符数一致 |
| `context` | string | ✓ | 词汇类别/上下文（如"人名"、"地名"、"官职"） |
| `note` | string | ✓ | 详细注释，说明特殊读音原因 |

### 拼音格式要求

- 使用带声调的拼音（如 mò, dú, chán, yú）
- 声调符号标在韵母上（ā á ǎ à）
- 每个汉字对应一个拼音
- `pinyin` 数组长度必须等于 `text` 字符串长度

## 匹配规则

### 优先级

1. **最长匹配优先**: "冒顿单于"（4字）优先于"冒顿"（2字）
2. **从左到右扫描**: 按文本顺序依次匹配
3. **完全匹配**: 只有完整匹配才生效，不支持部分匹配

### 匹配示例

```
文本: "冒顿单于攻破月氏"

匹配过程:
1. 检查"冒顿单于" (4字) ✓ 匹配成功，应用特殊读音
2. 跳过已匹配的4个字
3. 继续处理"攻破月氏"，使用标准读音
```

## 添加新词条

### 添加流程

1. **确认特殊读音**: 查阅权威古籍注音、学术资料
2. **编辑JSON文件**: 在 `docs/data/special-pronunciation.json` 中添加条目
3. **遵循格式规范**: 确保字段完整、拼音正确
4. **按长度排序**: 将新词条插入到合适位置（长词在前）
5. **更新版本号**: 修改 `lastUpdated` 字段
6. **测试验证**: 生成包含该词的HTML页面，验证读音正确

### 添加示例

假设要添加"大宛"的特殊读音：

```json
{
  "text": "大宛",
  "pinyin": ["dà", "yuān"],
  "context": "西域国名",
  "note": "'宛'在国名中读yuān，不读wǎn"
}
```

## 词条维护准则

### 收录标准

✅ **应该收录**:
- 古代专有名词的特殊读音
- 与现代读音差异显著的词汇
- 容易读错的人名、地名
- 姓氏、官职、族群称号的特殊读法

❌ **不应收录**:
- 现代标准读音
- 仅有声调差异的多音字（由拼音库自动处理）
- 方言读音
- 过于生僻的词汇

### 注释规范

- **简明扼要**: 一句话说明特殊读音原因
- **标注差异**: 指出与常见读音的不同
- **提供上下文**: 说明词汇使用场景

#### 好的注释示例
```
"'冒'读mò，'顿'读dú，匈奴单于名"
"'会'在地名中读kuài，不读huì"
```

#### 不好的注释示例
```
"这个词读音特殊"  // 太笼统
"读音见《说文解字》"  // 不直观
```

### 加载机制

脚本启动时自动加载词表：
```javascript

```

### 匹配算法

```javascript
function matchSpecialPronunciation(text, startIndex) {
  // 从长到短遍历词表
  for (let entry of specialPronunciations) {
    const substr = text.substring(startIndex, startIndex + entry.text.length);
    if (substr === entry.text) {
      return entry;  // 返回匹配结果
    }
  }
  return null;  // 无匹配
}
```

## 故障排除

### 问题1: 特殊读音不生效

**可能原因**:
- JSON格式错误
- 词表加载失败
- 拼音数组长度不匹配

**解决方法**:
1. 检查浏览器控制台错误信息
2. 验证JSON格式：使用在线JSON验证工具
3. 确认`pinyin`数组长度等于`text`长度

### 问题2: 部分字使用特殊读音，部分字不使用

**原因**: 匹配了较长的词，但只有部分字在词表中

**解决方法**: 确保词表中包含所有相关长度的词条

示例：
```json
// 同时包含长短词条
{"text": "冒顿单于", "pinyin": ["mò", "dú", "chán", "yú"], ...},
{"text": "冒顿", "pinyin": ["mò", "dú"], ...},
{"text": "单于", "pinyin": ["chán", "yú"], ...}
```

### 问题3: 词表文件404

**原因**: 相对路径不正确

**解决方法**:
- 确认HTML文件位于 `docs/chapters/` 目录
- 确认JSON文件位于 `docs/data/` 目录
- 相对路径应为 `../data/special-pronunciation.json`
