# 常识知识库索引 (Common Sense Knowledge Base)

> **目的**: 收集《史记》时代的普适性规律知识,为推理、检测、验证提供基准。

**原则**:
- ✅ 只收录**普适性规律**,不描述具体人和事
- ✅ 每条知识必须标注**时代范围**和**适用条件**
- ✅ 优先收录可量化、可验证的知识
- ✅ 明确标注数据来源和置信度

## 知识提取流程

```
原始史料/研究成果
    ↓ 阅读分析
提取候选知识点
    ↓ 普适性筛选 (是否适用于多个案例?)
    ↓ 可验证性检查 (是否有数据支撑?)
规范化表述
    ↓ 时代标注 (先秦/秦/汉)
    ↓ 来源标注 (文献/学界共识/推测)
    ↓ 置信度标注 (高/中/低)
录入知识库
    ↓ 版本控制
定期审核与更新
```

## 维护机制

### 1. 定期审核

- **季度审核**: 检查已有知识是否需更新
- **年度审核**: 整合新研究成果

### 2. 版本管理

- **小版本** (v1.1, v1.2): 新增条目、修正错误
- **大版本** (v2.0): 结构性重组、大量新增

### 3. 质量控制

- 每条新增知识必须标注来源
- 争议性知识必须标注"存疑"
- 定期与学界最新研究对照

### 4. 协作机制

- 提交新知识 → 审核 → 录入
- 发现错误 → Issue → 修正
- 重大更新 → 讨论 → 投票 → 执行

## 贡献指南

### 如何贡献新知识

1. **选择合适的知识库文件** (或创建新文件)
2. **按照规范格式** 编写知识条目
3. **标注数据来源** 和置信度
4. **提交Pull Request** 或Issue
5. **等待审核** 和讨论

### 如何报告错误

1. 在GitHub Issue中描述错误
2. 提供正确的数据来源
3. 说明影响范围
