# 矛盾检测方法论

> 《史记》跨章节矛盾自动化检测的系统方法

---

## 一、核心思路

《史记》130篇中，许多历史事件在多个篇章中都有记载（如楚汉相争同时记于007项羽本纪、008高祖本纪）。这些重叠记载提供了交叉验证的机会，也是矛盾检测的主战场。

**基本原则**：
1. 同一事件在不同章节的记载应该一致（事实）
2. 不一致之处可能是：①矛盾 ②视角差异 ③详略不同
3. 矛盾是史源批判和真实性推理的重要线索

---

## 二、矛盾类型

基于SKILL_07的分类体系：

### 2.1 DATE（日期矛盾）

**定义**：同一事件在不同章节记载的时间不一致。

**检测规则**：
- 同一事件的公元纪年不同
- 同一事件的月份不同
- 同一事件的季节不同

**已发现案例**：
| 事件 | 章节A | 章节B | 时间差 | 文件 |
|------|-------|-------|--------|------|
| 陈涉起义 | 007项羽本纪: 七月 | 008高祖本纪: 秋 | 表述不一致 | [contradiction_007_008.md](contradiction_007_008.md) |
| 太子丹之死 | 086刺客列传: 前222年 | 006秦始皇本纪: 前226年 | 4年 | [contradiction_086_006.md](contradiction_086_006.md) |
| 下宫之难 | 039晋世家: 前583年 | 043赵世家: 前597年 | 14年 | [contradiction_039_043.md](contradiction_039_043.md) |

**自动化程度**：高（★★★★★）
- 可从事件索引的公元纪年字段直接比对
- 月份/季节差异需要解析时间字段

---

### 2.2 DETAIL（细节矛盾）

**定义**：同一事件的数量、地点、参与者等细节不一致。

**检测规则**：
- 数量矛盾：兵力/斩首/距离等数字差异>10倍
- 地点矛盾：同一事件的发生地不一致
- 人物矛盾：同一事件的参与者集合不一致

**已发现案例**：
| 事件 | 维度 | 章节A | 章节B | 差异 |
|------|------|-------|-------|------|
| 东城快战 | 斩首数 | 007: 数十百人 | 008: 八万 | 差异巨大（可能统计口径不同） |
| 攻城阳战役 | 地点数 | 007: 5处 | 008: 2处 | 007更完整 |
| 垓下参战力量 | 描述 | 007: 多方汇聚 | 008: 韩信三十万 | 强调重点不同 |

**自动化程度**：中高（★★★★☆）
- 数量矛盾：可从事件索引的数字字段提取比对
- 地点/人物矛盾：需要NER+实体对齐

---

### 2.3 ATTRIBUTION（归因矛盾）

**定义**：同一事件的责任归属、功劳归属不同。

**检测规则**：
- 行为主体不同（谁做的？）
- 决策者不同（谁决定的？）
- 受益者不同（谁得利？）

**已发现案例**：
| 事件 | 维度 | 章节A | 章节B | 差异原因 |
|------|------|-------|-------|---------|
| 函谷关守关 | 守关者 | 007: 关守卒 | 008: 沛公 | 视角差异（淡化 vs 强调） |
| 鸿门宴核心推动者 | 叙事重心 | 007: 范增/项庄/项伯 | 008: 张良/樊哙 | 不同阵营视角 |

**自动化程度**：低（★★☆☆☆）
- 需要理解叙事语境
- 需要LLM判断责任归属

---

### 2.4 SEQUENCE（顺序矛盾）

**定义**：事件顺序或因果链在不同章节的记载不一致。

**检测规则**：
- 时间线顺序不同（A→B vs B→A）
- 因果链不同（A导致B vs C导致B）
- 事件链完整性不同（A→C vs A→B→C）

**已发现案例**：
| 事件 | 维度 | 章节A | 章节B | 差异 |
|------|------|-------|-------|------|
| 下宫之难 | 时间+因果 | 039: 前583年直接诛杀 | 043: 前597年诛杀→15年→复位 | 因果链完整性不同 |

**自动化程度**：中（★★★☆☆）
- 需要事件链建模
- 需要因果关系推理

## 三、检测流程

### 3.1 事件对齐（Event Alignment）

**目标**：找出在两个章节中都有记载的"同一事件"。

**方法1：公元纪年匹配**
```python
# 从事件索引中提取公元年份
events_A = get_events_from_chapter("007")
events_B = get_events_from_chapter("008")

# 找出公元纪年相同或相近（±1年）的事件对
overlapping_events = []
for event_a in events_A:
    for event_b in events_B:
        if abs(event_a.year - event_b.year) <= 1:
            overlapping_events.append((event_a, event_b))
```

**方法2：实体共现匹配**
```python
# 提取事件参与的核心实体
entities_a = extract_entities(event_a)
entities_b = extract_entities(event_b)

# 计算实体交集
common_entities = entities_a & entities_b

# 如果交集>阈值，认为是同一事件
if len(common_entities) >= 2:
    return True
```

**方法3：关键词匹配**
```python
# 提取事件描述的关键词
keywords_a = extract_keywords(event_a.description)
keywords_b = extract_keywords(event_b.description)

# 计算语义相似度
similarity = compute_similarity(keywords_a, keywords_b)

# 如果相似度>阈值，认为是同一事件
if similarity >= 0.7:
    return True
```

**综合判断**：
```
IF (公元纪年匹配 OR 实体共现匹配) AND 关键词相似度>0.5:
    认为是同一事件
```

---

### 3.2 逐维度对比

对已对齐的事件对，逐维度比对：

| 维度 | 提取字段 | 比对方法 | 输出 |
|------|---------|---------|------|
| 时间 | 公元年/月/季 | 精确匹配 | 是否一致 |
| 地点 | 地名实体列表 | 集合比对 | 差异列表 |
| 人物 | 人物实体列表 | 集合比对 | 差异列表 |
| 数量 | 数字提取 | 数值比对 | 差异比例 |
| 归因 | 主语/谓语提取 | LLM判断 | 归因差异 |
| 顺序 | 事件链 | 序列比对 | 顺序差异 |

---

### 3.3 矛盾分类

**判断规则**：

```
IF 时间差异 AND 无法用纪年方式解释:
    TYPE = DATE矛盾

IF 数字差异>10倍 AND 无法用统计口径解释:
    TYPE = DETAIL矛盾（数量）

IF 地点集合不同 AND 不是详略差异:
    TYPE = DETAIL矛盾（地点）

IF 归因主体不同 AND 不是视角差异:
    TYPE = ATTRIBUTION矛盾

IF 事件顺序不同 OR 因果链不同:
    TYPE = SEQUENCE矛盾

IF 只是详略不同 BUT 无事实矛盾:
    TYPE = 视角差异（非矛盾）
```

## 四、自动化实现路径

### 4.1 半自动化流程（当前可行）

```
步骤1: 人工选择章节对（如007+008）
    ↓
步骤2: 从事件索引中提取事件列表
    ↓
步骤3: 自动对齐（公元纪年+实体共现）
    ↓
步骤4: 自动比对（时间/数量/地点/人物）
    ↓
步骤5: LLM评估（归因/顺序/真矛盾vs视角差异）
    ↓
步骤6: 生成矛盾报告JSON
```

**工具需求**：
- 事件索引读取器
- 实体对齐算法
- 数字提取与比对
- LLM评估提示词

---

### 4.2 全自动化流程（未来目标）

```
步骤1: 扫描全书130篇，识别高重叠章节对
    ↓
步骤2: 批量事件对齐（基于知识图谱）
    ↓
步骤3: 批量矛盾检测（规则+LLM）
    ↓
步骤4: 矛盾分级（高/中/低研究价值）
    ↓
步骤5: 输出《史记矛盾图谱》可视化
```

**挑战**：
- 事件对齐的准确率（需要高质量知识图谱）
- 视角差异 vs 真矛盾的判断（需要深度理解）
- 矛盾价值评估（需要领域知识）

## 五、LLM评估提示词模板

### 5.1 归因矛盾判断

```
你是一位《史记》矛盾检测专家。请判断以下两段记载是否存在归因矛盾。

事件: {event_name}
公元年: {year}

章节A ({chapter_A}):
{text_A}

章节B ({chapter_B}):
{text_B}

请判断：
1. 两段记载的核心行为主体是否相同？（是/否/存疑）
2. 如果不同，这是否构成归因矛盾？（是/否）
3. 还是只是视角差异？（是/否）
4. 简要说明理由（1-2句话）

输出格式:
{
  "主体相同": true/false,
  "归因矛盾": true/false,
  "视角差异": true/false,
  "理由": "..."
}
```

### 5.2 矛盾价值评估

```
你是一位历史研究助手。请评估以下矛盾的研究价值。

矛盾类型: {type}
事件: {event}
章节对: {chapters}
差异描述: {difference}

请判断：
1. 这个矛盾是否有助于史源批判？（高/中/低）
2. 这个矛盾是否揭示了历史真相的线索？（是/否）
3. 这个矛盾是否值得深入研究？（高/中/低价值）
4. 推荐的研究方向（1句话）

输出格式:
{
  "史源批判价值": "高/中/低",
  "真相线索": true/false,
  "研究价值": "高/中/低",
  "研究方向": "..."
}
```
