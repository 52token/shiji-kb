# SKILL 01a: 标注完整性维护 — 标注铁律的守护者

> **标注铁律**：标注工作只能添加`〖TYPE〗`标记符号，不得对原文字符做任何修改（增删改）。

---

## 一、铁律的重要性

### 1.1 为什么需要铁律

标注的本质是**语义标记**，而非**文本编辑**：

```
原文:  五子作歌，太康失政。
标注:  〖$五〗子作歌，〖@太康〗失政。
去标:  五子作歌，太康失政。  ← 必须与原文逐字相同
```

**违反铁律的后果**：
- ❌ 实体索引基于错误文本，指向不存在的词句
- ❌ 事件抽取对应不到原文段落
- ❌ 三家注锚点失效
- ❌ 读者对照原书时发现文字不一致，丧失信任

### 1.2 完整性的定义

**完整性** = 去除全部标注符号后，所得纯文本与 `archive/chapter/*.txt` 原始底本逐字相同

```
去除对象：
- Markdown标题 (# ### ...)
- 实体标注 (〖@人名〗 〖=地名〗 ...)
- 动词标注 (⟦◈伐⟧ ⟦◉诛⟧ ...)
- 语义区块 (::: 围栏)
- 段落编号 ([1] [1.1] ...)
- 列表符号 (- 项)
- 引用符号 (> 文字)
- 粗体 (**文字**)
- 分隔线 (---)
```

## 二、工具链

### 2.1 完整性检查脚本

```bash
# 检查全部130章
python scripts/lint_text_integrity.py

# 检查指定章节
python scripts/lint_text_integrity.py 002 005 013

# 显示全部差异（包括标点/编码变体）
python scripts/lint_text_integrity.py --all
```

**输出格式**：

```
  ✓ 001_五帝本纪: 0处实质差异  28处标点  1处编码
  ✗ 002_夏本纪: 1处实质差异  49处标点  2处编码
  ✓ 003_殷本纪: 0处实质差异  12处标点  0处编码
```

报告保存到 `logs/lint_text_integrity.txt`（全部章节）或 `logs/lint_text_integrity_XXX.txt`（指定章节）。

### 2.2 差异分类

脚本自动将差异分为三类：

| 分类 | 说明 | 是否需要修复 |
|------|------|--------------|
| **实质差异** | 汉字字符被增加/删除/改写 | ✅ 必须修复（P0） |
| **标点规范化** | 全角↔半角标点（，→, 等） | ❌ 可忽略 |
| **编码规范化** | PUA私用区字符→标准Unicode | ❌ 可忽略 |

### 2.3 规范化映射

脚本通过 `_NORM_TABLE` 消除等价变体差异：

```python
_NORM_TABLE = {
    # 引号全部删除（标注规范：原文无引号时标注可添加全角引号）
    '\u201c': '',  # "
    '\u201d': '',  # "
    # 繁简变体
    '\u540e': '\u5f8c',  # 后 → 後
    '\u4e8e': '\u65bc',  # 于 → 於
    # 通假字
    '\u8c37': '\u7a40',  # 谷 → 穀
    ...
}
```

**规范化后仍存在的差异** → 实质差异，需要修复。

## 三、修复流程

### 3.1 典型案例：002_夏本纪

**Lint报告**：

```
=== 002_夏本纪 ===  实质:1  标点:19  编码:2
  ■ 实质差异 1 处：
  [1] 替换
      原文  : '作'
      标注  : '之'
      原文上下文 : …五子作歌，太康失政…
      标注上下文 : …五子之歌，太康失政…
```

**分析**：

1. 查看原始底本 `archive/chapter/002_夏本纪.txt`（**主参照**）：
   - 第32行（正文）：`作五子之歌`
   - 第37行（太史公曰）：`五子作歌，太康失政` ✓

2. 查看标注文件 `chapter_md/002_夏本纪.tagged.md:195`：
   - `〖{五子之歌〗，〖@太康〗失政。` ✗

3. 参照维基文库版本（辅助验证）：
   - `archive/wikisource_shiji/002_夏本紀.html`: 正文为`作五子之歌`，但太史公论中无此句（维基版本太史公论不完整）
   - 确认：原始底本正确

**修复**：

```bash
# 在 chapter_md/002_夏本纪.tagged.md:195 修改
修改前:  〖{五子之歌〗，〖@太康〗失政。
修改后:  〖$五〗子作歌，〖@太康〗失政。
```

**验证**：

```bash
python scripts/lint_text_integrity.py 002
# 输出: ✓ 002_夏本纪: 0处实质差异  49处标点  2处编码
```

### 3.2 修复原则

1. **原始底本为准** — lint校验以 `archive/chapter/*.txt` 为唯一基准
2. **参照版本辅助** — 遇到疑问时参考 `archive/wikisource_shiji/` (维基文库)
3. **保留标注** — 修复时尽量保留已有的实体标注，只调整文字部分
4. **全章验证** — 修复后立即运行lint验证该章节
5. **底本有误时** — 先使标注与底本一致（通过lint），再记录校勘意见

### 3.3 常见错误类型

| 错误类型 | 示例 | 修复方法 |
|---------|------|---------|
| **擅自改字** | `作`→`之` | 恢复原字，删除多余标注 |
| **擅自加字** | 原文`王冠`→标注`王冠礼` | 删除多余的`礼`字 |
| **擅自删字** | 原文`公子婴`→标注`子婴` | 补回缺失的`公`字 |
| **标点转换** | 原文`。`→标注`，` | 恢复原标点（全角句号） |
| **引号添加** | 原文无引号→标注加引号 | 删除引号（或保留，属于标点规范化） |

## 四、白名单机制

### 4.1 作用

已确认的**正确校勘**记录到白名单，lint时自动跳过，避免误报。

**场景举例**：
- 原文底本有误（如`？`缺字），标注时已根据权威版本补正
- 原文有衍文，标注时已剔除

### 4.2 格式

文件路径：`scripts/lint_whitelist.txt`

```
# 格式：章节号<TAB>原文片段<TAB>标注片段<TAB>说明
004	？	历	补缺字：季？→季历
005	。	 	删除衍文句号
006	公	 	删除多余"公"字（原文衍文）
```

### 4.3 使用方法

1. 确认差异是**正确的校勘**（而非标注错误）
2. 在 `scripts/lint_whitelist.txt` 添加一行
3. 再次运行lint，该差异标记为"校勘"

```bash
python scripts/lint_text_integrity.py 004
# 输出: ✓ 004_周本纪: 0处实质差异  41处标点  1处编码  校勘:3
```

## 五、参照资源

### 5.1 版本优先级

| 优先级 | 资源 | 路径 | 说明 |
|-------|------|------|-----|
| 1 | **原始底本（archive/chapter）** | `archive/chapter/*.txt` | **主参照**：lint校验基准，标注文件必须与之逐字相同 |
| 2 | **维基文库史记** | `archive/wikisource_shiji/*.html` | 繁体，基于中华书局本，校勘质量高，用于疑难处参照 |
| 3 | **其他数字化版本** | ctext.org, 国学网 | 在线查阅，辅助参照 |
| 4 | **中华书局点校本** | 实体书/PDF | 权威终极参照 |

**重要**：完整性检查以 `archive/chapter/*.txt` 为基准。如果发现原始底本有误，应：
1. 先修复标注文件，使其与 `archive/chapter/*.txt` 一致（通过lint检查）
2. 然后在 `archive/chapter/*.txt` 中记录校勘意见（注释形式）
3. 最后在多版本互校流程中统一修正底本

### 5.2 查找方法

**示例：查找"五子作歌"在各版本中的原文**

```bash
# 查找原始底本（主参照）
grep "五子" archive/chapter/002_夏本纪.txt

# 查找维基文库版本（辅助参照）
grep -o "五子[^<]*歌" archive/wikisource_shiji/002_夏本紀.html

# 查看完整上下文（-C10 = 上下各10行）
grep -C10 "五子" archive/wikisource_shiji/002_夏本紀.html
```

### 5.3 HTML提取技巧

维基文库HTML中，文本内容在 `<p>` 标签中，去除HTML标签后即为纯文本：

```bash
# 提取纯文本（去除HTML标签）
sed 's/<[^>]*>//g' archive/wikisource_shiji/002_夏本紀.html | grep "五子"
```

4. **记录白名单**：遇到正确校勘时添加到白名单
5. **批量验证**：完成一批后运行全章节lint

## 八、与父SKILL的关系

本SKILL（01a标注完整性维护）是 [SKILL_01_古籍校勘](SKILL_01_古籍校勘.md) 的子技能：

```
SKILL_01 古籍校勘 (archive/chapter/*.txt 原始底本校正)
    ├─ 校勘原则
    ├─ 校勘来源（中华书局本/维基文库...）
    ├─ 常见错误类型
    └─ ↓ 衍生子技能
       SKILL_01a 标注完整性维护 (chapter_md/*.tagged.md 标注后验证)
           ├─ 标注铁律
           ├─ 完整性检查
           ├─ 差异分类
           └─ 修复流程
```

**分工**：
- **SKILL_01**：底本阶段，在 `archive/chapter/` 修正网络来源txt的错误，建立可信原始底本
- **SKILL_01a**：标注阶段，确保标注过程中没有违反铁律，擅自修改原文
- **说明**：`docs/original_text/` 是供读者查看的原文版本，理论上应与 `archive/chapter/` 保持一致

## 十、附录：差异类型速查表

| lint输出 | 含义 | 常见原因 | 修复方法 |
|---------|------|---------|---------|
| **替换** | 原文字符被改写 | 标注时误改（作→之） | 恢复原字 |
| **插入（标注多字）** | 标注文件多出字符 | 擅自加字（王冠→王冠礼） | 删除多余字符 |
| **删除（标注少字）** | 标注文件少字符 | 标注时丢失（公子婴→子婴） | 补回缺失字符 |
| **标点规范化** | 全角半角转换 | 历史遗留，可忽略 | 不修复（或统一为全角） |
| **编码规范化** | PUA字符差异 | 原文PUA→标注标准Unicode | 不修复（已规范化） |
